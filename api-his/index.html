HISinOne-Funktionen

Diese Seite beschreibt die Funktionen aus hexscrapepy.scraper. Der Fokus liegt auf browsergesteuertem Scraping in HISinOne: erst Basisdaten laden, dann pro Veranstaltung die Detailseite öffnen und weiter auslesen.

Rollen der Funktionen

Öffentlich empfohlen

Funktion Rolle Kurzbeschreibung
scrape_all_sem Wrapper führt den kompletten HISinOne-Workflow aus, also Basisdaten und Detaildaten für ein oder mehrere Semester
fetch_base_data Basisdaten lädt die Trefferliste eines Semesters aus dem HIS und speichert sie als CSV-Datei
scrape_data_playwright Detaildaten öffnet für jede Veranstaltung die Detailseite und ergänzt Termine, Inhalte sowie Modul- und Studiengangsdaten
start_his_session Session startet Playwright, öffnet Chromium und lädt die HISinOne-Startseite

Modular nutzbar

Funktion Rolle Kurzbeschreibung
his_select_semester_base_data Navigation wählt ein Semester aus und startet direkt die Suche für die Kursübersicht
his_select_semester Navigation setzt das Semester in der Suchmaske, ohne sofort eine Suche auszulösen
his_search_and_get_data Extraktion liest die sichtbare Ergebnistabelle über alle Trefferseiten hinweg in ein gemeinsames DataFrame ein
scrape_termine Extraktion liest den Terminblock einer HIS-Detailseite als Label-Wert-Paare aus
scrape_inhalte_playwright Extraktion liest die Textblöcke aus dem Tab Inhalte und macht daraus Spalten einer Ergebniszeile
scrape_module_studiengaenge_playwright Extraktion liest die Tabellen zu Modulen und Studiengängen aus der Detailseite aus

Intern oder technisch

Funktion Rolle Kurzbeschreibung
sanitize_semester_name Helper wandelt Semesternamen in kurze, dateitaugliche Kennungen wie wise2024 um
check_dependencies Helper prüft, ob die wichtigsten Python-Abhängigkeiten für den HIS-Lauf importierbar sind
clean_prefixes Helper entfernt wiederholte Spaltenüberschriften aus Tabellenzellen
clean_titles Helper vereinheitlicht Veranstaltungstitel für Suche und Resume-Abgleich
normalize_his_text Helper normalisiert Texte aus dem HIS für robuste Stringvergleiche

Typischer Ablauf

Ein vollständiger HISinOne-Lauf sieht meist so aus:

  1. Mit fetch_base_data() oder direkt mit scrape_all_sem() die Veranstaltungsliste eines Semesters erzeugen.
  2. Die Basisdatei kurz prüfen, zum Beispiel auf Spaltennamen und sichtbare Trefferqualität.
  3. Mit scrape_data_playwright() die Detailseiten der gefundenen Veranstaltungen nachziehen.
  4. Innerhalb dieses Detail-Scrapings verarbeiten die kleineren Extraktionsfunktionen die einzelnen Bereiche der Kursseite.

fetch_base_data(url, export_path, semester_name='Sommersemester 2025', headless=False)

Lädt die Übersicht aller Kurse für ein Semester herunter und speichert sie als CSV.

Diese Funktion ist der typische Startpunkt: Sie lädt die Übersichtstabelle für ein Semester und speichert sie als Datei.

Parameter

url (str): Start-URL der Kursübersicht. export_path (str): Speicherort für die CSV-Datei. semester_name (str): Name des Semesters. headless (bool): Unsichtbarer Browser?

Rückgabe

pd.DataFrame: Tabelle mit allen Kursen.

Beispiel

fetch_base_data(url, "daten.csv", "Sommersemester 2025")

sanitize_semester_name(semester_name)

Create a short, filesystem-safe token from a semester label.

The function recognizes common German winter and summer semester patterns and converts them into compact identifiers such as wise2024 or sose2025. If no known pattern is found, it falls back to a generic alphanumeric slug.

Parameters:

Name Type Description Default
semester_name str

Original semester label.

 required

Returns:

Type Description
str

Short semester token suitable for filenames and folder names.

Run the complete HISinOne workflow for one or more semesters.

For each semester the function first ensures that base course data exists as CSV and then enriches those rows with detail data scraped from the course pages. This is the main high-level entry point for HISinOne sources.

Öffentliche Paket-API

Diese API wird über hexscrapepy direkt exportiert:

hexscrapepy

HEXscrapePy package.

Die öffentliche API wird per Lazy-Import bereitgestellt. Dadurch bleibt ein einfaches import hexscrapepy leichtgewichtig und zieht optionale schwerere Abhängigkeiten wie Playwright erst dann nach, wenn eine entsprechende Funktion wirklich genutzt wird.
