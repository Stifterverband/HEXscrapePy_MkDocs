Schnellstart

Diese Dokumentation ist in drei Ebenen aufgebaut:

  1. Start: Was das Paket grundsätzlich kann.
  2. Workflows: Wie ein typischer Scraping-Lauf für HISinOne oder LSF aussieht.
  3. API: Welche Funktionen es gibt und welche Aufgabe jede Funktion im Detail übernimmt.

Für wen ist HEXscrapePy gedacht?

HEXscrapePy ist ein Python-Paket zum automatisierten Abrufen von Lehrveranstaltungsdaten aus Hochschulsystemen. Im Repository gibt es aktuell zwei große Funktionsblöcke:

  • hexscrapepy.scraper für HISinOne/HIS-Systeme mit browserbasiertem Scraping via Playwright
  • hexscrapepy.lsf_scraper für LSF-Systeme mit einer Mischung aus Browser-Navigation, HTML-Parsing und HTTP-Abruf

Empfohlener Einstieg

Wenn du das Paket zum ersten Mal verwendest, starte nicht mit den niedrigsten Hilfsfunktionen, sondern mit den Wrappern:

  • HISinOne: scrape_all_sem()
  • LSF: scrape_lsf_all_semesters()

Diese beiden Funktionen koordinieren den größten Teil des Workflows bereits selbst:

  • Auswahl von Semestern
  • Laden oder Erstellen von Basisdaten
  • Detail-Scraping
  • Zwischenspeicherung
  • Ausgabe in Dateien

Typische Arbeitsweise

Es gibt zwei sinnvolle Nutzungsmodi:

1. Einfache Nutzung über Wrapper

Geeignet, wenn du komplette Semester oder komplette Hochschulen scrapen möchtest.

2. Modulare Nutzung über Einzelfunktionen

Geeignet, wenn du:

  • nur einen Teil des Workflows brauchst
  • einzelne Parser testen willst
  • HTML oder Tabellen separat weiterverarbeiten möchtest
  • Fehler im Ablauf gezielt debuggen willst

Leitidee der API

Die Funktionen im Paket lassen sich grob in vier Gruppen einteilen:

  • Orchestrierung: startet komplette End-to-End-Läufe
  • Navigation und Suche: bewegt sich durch die Zielsysteme
  • Extraktion: liest Tabellen, Textfelder und Metadaten aus
  • Hilfsfunktionen: normalisiert Texte, Dateinamen oder Resume-Daten

In den API-Seiten ist deshalb jeweils angegeben:

  • wofür die Funktion gedacht ist
  • in welchem Schritt des Workflows sie auftaucht
  • ob sie eher öffentlich nutzbar oder intern/technisch ist

Öffentliche Einstiegspunkte

Die öffentliche API wird über hexscrapepy.__init__ bereitgestellt. Besonders relevant sind:

  • scrape_all_sem
  • fetch_base_data
  • start_his_session
  • scrape_base_data_lsf
  • scrape_lsf_veranstaltungen
  • scrape_lsf_all_semesters

Wenn du nachvollziehen willst, was intern passiert, findest du die restlichen Helfer in den API-Seiten zu HISinOne und LSF.
