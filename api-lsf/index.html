LSF-Funktionen

Diese Seite beschreibt die Funktionen aus hexscrapepy.lsf_scraper. Der Schwerpunkt liegt auf LSF-Systemen: zuerst eine Veranstaltungsliste aufbauen, dann die Detailseiten laden und in einzelne Inhaltsblöcke zerlegen.

Rollen der Funktionen

Öffentlich empfohlen

Funktion Rolle Kurzbeschreibung
lsf_scrape_base_data Basisdaten erstellt eine Veranstaltungsliste für ein oder mehrere Semester und schreibt sie als Basisdatei
lsf_scrape_veranstaltungsdetails Detaildaten lädt für vorhandene URL_Event-Links die Detailseiten und ergänzt die inhaltlichen Zusatzfelder
lsf_scrape_all_semesters Wrapper kombiniert Basislauf und Detail-Scraping zu einem kompletten LSF-Workflow pro Semester

Modular nutzbar

Funktion Rolle Kurzbeschreibung
lsf_select_semester Navigation wechselt im LSF zum gewünschten Semester und öffnet das Vorlesungsverzeichnis
lsf_extract_veranstaltungstitel Extraktion liest den sichtbaren Veranstaltungstitel aus der Detailseite
lsf_extract_grunddaten Extraktion liest den Block Grunddaten zur Veranstaltung als strukturierte Felder aus
lsf_extract_termine Extraktion liest alle Terminzeilen einer Veranstaltung in Tabellenform aus
lsf_extract_dozenten Extraktion extrahiert die Tabelle der durchführenden Dozentinnen und Dozenten
lsf_extract_zugeordnete_personen Extraktion extrahiert weitere zugeordnete oder verantwortliche Personen als verschachtelte Tabelle
lsf_extract_studiengaenge Extraktion liest den Studiengangsblock als verschachtelte Tabelle aus
lsf_extract_module Extraktion liest den Modulblock als verschachtelte Tabelle aus
lsf_extract_einrichtung Extraktion liest organisatorische Zuordnungen oder Einrichtungen aus
lsf_extract_weitere_angaben Extraktion liest den Block Weitere Angaben zur Veranstaltung generisch als Felder aus
lsf_extract_lerninhalt Extraktion zieht das Feld Lerninhalte aus den weiteren Angaben heraus
lsf_extract_kommentar Extraktion zieht das Feld Kommentar aus den weiteren Angaben heraus
lsf_fetch_course_html Abruf lädt das rohe HTML einer LSF-Detailseite per HTTP

Intern oder technisch

Funktion Rolle Kurzbeschreibung
lsf_clean_text Helper bereinigt Freitext und ersetzt leere Werte durch einen einheitlichen Platzhalter
lsf_normalize_url Helper reduziert LSF-Navigations-URLs auf stabile Vergleichswerte für Duplikaterkennung
_scrape_via_path Workflow interner Basislauf über die Baumstruktur des Vorlesungsverzeichnisses
_scrape_via_search Workflow interner Basislauf über die Suchmaske mit Pagination
_find_event_tables Parser erkennt Tabellen, die tatsächlich Veranstaltungslisten enthalten
_parse_event_table Parser liest einzelne Veranstaltungszeilen in Ergebnis-Dictionaries ein
_find_lsf_section_table Parser findet Bereichstabellen robust über summary-Texte oder Container-IDs
_needs_expanded_lsf_fetch Steuerung entscheidet, ob eine Detailseite noch einmal mit aufgeklappten Bereichen geladen werden sollte
_load_lsf_results_json Resume lädt vorhandene Detailergebnisse für Resume-Läufe
_write_lsf_results_json Resume schreibt Zwischenergebnisse als JSON-Datei zurück
_build_lsf_detail_frame Merge reduziert gescrapte Detaildaten auf die Spalten, die später sauber gemergt werden können

Typischer Ablauf

Ein vollständiger LSF-Lauf sieht meist so aus:

  1. Mit lsf_scrape_all_semesters() oder manuell mit lsf_scrape_base_data() zunächst die Veranstaltungsliste eines Semesters aufbauen.
  2. Auf Basis der erzeugten URL_Event-Spalte die Detailseiten mit lsf_scrape_veranstaltungsdetails() abrufen.
  3. Innerhalb des Detail-Scrapings lesen die einzelnen extract_*-Funktionen die jeweiligen Inhaltsblöcke aus dem HTML.
  4. Die Ergebnisse werden anschließend zu einem gemeinsamen DataFrame pro Semester zusammengeführt.

lsf_extract_weitere_angaben(soup)

Extrahiert alle Felder aus Weitere Angaben zur Veranstaltung.

Die LSF-Ansicht kapselt diesen Bereich teils in auf-/zuklappbare Container (z.B. veranst_slide_inhalt). Für das Parsing ist das unkritisch, solange die Tabelle im HTML enthalten ist. Diese Funktion liest deshalb die Tabellenzeilen generisch aus und liefert eine einzige Zeile mit allen vorhandenen Feldern zurück.

lsf_extract_zugeordnete_personen(soup)

Extrahiert die zugeordneten Personen als verschachtelte Tabelle.

Diese Funktion ist für LSF-Blöcke wie Zugeordnete Person oder Verantwortliche Dozenten gedacht. Das Ergebnis wird bewusst als verschachtelte Tabelle zurückgegeben, damit die Struktur aus dem HTML möglichst erhalten bleibt.

Rückgabespalten der inneren Tabelle: - zugeordnete_person - zustaendigkeit

Lade eine LSF-Detailseite mit aufgeklappten Toggle-Bereichen.

Öffentliche Paket-API

Diese API wird über hexscrapepy direkt exportiert:

hexscrapepy

HEXscrapePy package.

Die öffentliche API wird per Lazy-Import bereitgestellt. Dadurch bleibt ein einfaches import hexscrapepy leichtgewichtig und zieht optionale schwerere Abhängigkeiten wie Playwright erst dann nach, wenn eine entsprechende Funktion wirklich genutzt wird.
